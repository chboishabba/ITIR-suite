# Suite P0 Moonshot: Normalized Concepts

Date: 2026-04-02

## Purpose

Promote a new P0 suite-level moonshot above the current lane plans.

The current legal/compiler, state, capture, archive, and operator plans remain
useful, but they are stepping stones. They are not the top-level goal.

The broader target returns to the original smart-journal trajectory:

- capture difficult reality
- preserve it without losing provenance
- canonicalize it without silently rewriting it
- compile stable read-only state
- review and promote only what earns authority
- derive bounded operator products above that
- union those bounded products across domains, jurisdictions, languages, and
  time

## P0 Moonshot

The suite should become a normalized compiler for difficult reality rather
than a loose set of tools.

The normalized concepts are:

- source artifact
- provenance anchor
- context envelope
- canonical identity
- observed signal
- compiled state
- reviewable claim
- promoted record
- derived product
- follow obligation
- abstention / hold / unresolved pressure
- operator inspection surface
- bounded union surface

Current plans should be treated as local adopters of those concepts:

- capture/transcription lanes are source-artifact and provenance-anchor
  producers
- archive/chat/export lanes are canonical-identity and context-envelope
  producers
- state lanes are observed-signal to compiled-state reducers
- legal/reasoner lanes are reviewable-claim, promotion, abstention, and
  derived-product adopters
- operator/web lanes are inspection surfaces over compiled, promoted, and
  derived outputs
- research/retrieval lanes are bounded follow and artifact-generation
  adapters, not canonical truth producers

The suite priority order is therefore product-stack-first rather than
domain-first. Existing AU, GWB, Wikidata/Nat, Brexit, and similar
domain-specific plans remain useful adopters and proving grounds, but they do
not control the top-level order. The top-level order is driven by missing
stack segments in the correct owning repos:

- capture/archive producers
- state reducers
- review/promotion reducers
- operator/integration surfaces
- retrieval/bounded-follow adapters
- only then additional domain adopters where those stack contracts are real

## Constraints

Hard suite constraints for the P0 moonshot:

- no hidden rewrite of source reality
- no canonical truth without explicit promotion
- no second canonical reducer appearing in side systems
- no derived surface silently becoming authority
- no context-free consequential output
- no follow expansion without an explicit bounded trigger
- no panopticon drift

## Invariants

- provenance invariant:
  every consequential artifact can be traced to bounded source material or
  explicit promotion receipts
- normalization invariant:
  local adapters map into shared concepts instead of inventing silent
  incompatible categories
- expansion invariant:
  summaries and reduced views can expand back to anchors, IDs, and support
- context invariant:
  consequential artifacts preserve or reference a usable context envelope
- authority invariant:
  only explicit review/promotion creates truth-bearing canonical records
- replay invariant:
  the same bounded inputs and profile produce stable outputs
- derived-only invariant:
  graphs, rankings, comparisons, and operator overlays remain derived unless
  explicitly promoted
- append-only state invariant:
  historical state is preserved, not silently rewritten
- anti-panopticon invariant:
  broader context and better union never authorize covert identity stitching,
  person scoring, predictive judgment, or manipulative inference

## Preconditions

- source artifacts are addressable
- provenance anchors are sufficient for replay
- a context envelope exists wherever outputs would otherwise become detached
- each write path declares whether it is:
  - capture
  - archive
  - state compilation
  - review
  - promoted truth
  - derived inspection
  - bounded union
- any follow step declares its trigger, scope, and stop condition

## Postconditions

- outputs remain replayable and inspectable
- canonical writes carry promotion receipts
- derived outputs are explicitly marked as derived
- unresolved pressure remains visible as follow-needed, hold, or abstain
- operator surfaces can answer:
  - what is this
  - why does it exist
  - what supports it
  - what remains unresolved
- cross-project handoffs preserve IDs, lineage, and versioned assumptions

## Whole-Suite Flow

The suite-level end-state flow is:

1. capture or ingest bounded source artifacts
2. archive and canonicalize identity, time, anchors, and envelopes
3. emit observed signals and compile read-only state
4. extract reviewable claims where the domain requires structured judgment
5. follow justified dependencies in bounded ways
6. promote, abstain, or hold under explicit authority rules
7. derive packets, graphs, comparisons, dashboards, and operator views
8. union bounded promoted and derived products across domains where justified
9. let operators inspect, challenge, compare, and continue follow

The current P0 implication is:

- update root docs and roadmaps first
- only refine implementation where a submodule is clearly violating the new
  normalized shape

This is the new controlling suite-level moonshot note.
