# Global Authority Source Normalization

Date: 2026-04-03

## Purpose

Define the first bounded global-source widening rule above the current UK/EU
proving stack.

This is not a broad ingest mandate.

It is a contract and prioritization checkpoint for adding global authority
surfaces without breaking provenance, promotion boundaries, or bounded search.

## Current Decision

The immediate global-source pass should be:

- English-first operationally
- multilingual-aware structurally
- translation-bounded by governance

Plainly:

- prefer sources that already publish authoritative English text
- prefer sources with stable official parallel translations where English is
  one view, not the semantic center
- treat machine translation as an alignment and anomaly-detection aid only
- keep translation below promotion and outside canonical truth

## Source Selection Rule

Admit a source family into the next round when most of the following are true:

- deterministic identifier surface exists
- structured retrieval is available or HTML structure is stable enough
- provenance is preservable
- authority class is legible
- English is available directly or via stable official translation
- bounded follow can be defined without open-ended crawl behavior

## Priority Order

For the next widening round:

1. official English or English-available legal sources
2. official parallel multilingual sources with stable identifiers
3. evidentiary or archive surfaces after the normative layer is stable
4. lower-confidence multilingual portals only after a translation/alignment
   contract is implemented and proven safe

## Translation / Alignment Rule

Translation is not:

- extraction
- promotion
- canonical normalization

Translation is only:

- a weak alignment oracle
- a disagreement detector
- a bounded proposal surface

If translations disagree materially, the correct outcome is:

- uncertainty
- abstain
- follow-needed

not silent normalization.

## High-Value Exceptions

The following multilingual families are high-value early exceptions because
they are already known to publish stable parallel or well-translated material:

- UN Official Document System / UN Digital Library
- EUR-Lex
- similar treaty or transnational legal/document systems with stable official
  English views

These remain subject to the same normalized source contract and bounded follow
rules as the English-first families.

## Immediate Worker Read

The next bounded worker round should split into:

- normalized global source contract
- search/ref union plus authority-aware ranking
- bounded heterogeneous follow contract
- readiness metrics and invariants
- docs/control
- family ranking packet for the next non-UK/EU adopters

No lane should claim continent-complete implementation in this round.

## Constraints

- no fake global coverage claims
- no translation-derived truth
- no hidden second ontology centered on English
- no broad crawl behavior
- no source-family widening ahead of the normalized contract

## Invariants

- Search -> Identify -> Normalize -> Operate
- authority lattice remains explicit
- provenance remains first-class
- live vs fallback remains visible
- translation remains below promotion

## Preconditions

- current UK/EU proving path stays healthy
- normalized source contract exists or is being implemented in this round
- chosen sources can expose stable ids and provenance

## Postconditions

- new source families fit one normalized contract
- follow operators know the authority class they are traversing
- multilingual uncertainty remains visible instead of being flattened
- the current proof path remains non-regressed
