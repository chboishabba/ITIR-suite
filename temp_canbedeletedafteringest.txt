### Layer 2 — Projection layer

Everything user-facing derives from `ArticleState`.

There should be **three projections**, not three ingest systems.

#### A. Timeline projection

Purpose: produce date-bearing event rows.

Rule:

* only emit rows when anchor strength passes threshold,
* explicit dates > weak dates > heading/category/*VIBE* fallback. re vibe eg 'this looks like an event', and also ideally event predicate eg He woke up, brushed his teeth, and went to work... Should all be three separate things attributed to whoever we can reasonably determine 'he' to be, in correct order etc..

This preserves your current conservative timeline behaviour, but makes it clear that timeline is a **projection over article state**, not the main ingest ontology.

So:

* timeline rows are a filtered, anchored subset of extracted event candidates,
* no anchor = still kept in article state, just absent from timeline view.

That solves the current conceptual confusion.

#### B. Article-ingest projection

Purpose: whole-article coverage and AAO-style scoring.

This is the broadest projection:

* every sentence row,
* every extractable actor/action/object relation,
* claim-bearing and attribution-bearing structures,
* coverage metrics.

This is the canonical “what did we get from the article?” surface.

#### C. Revision projection

Purpose: compare two `ArticleState`s.

Important: revision should not compare “timeline rows” as its primary ontology.
It should compare canonical state first, then optionally report projection-level deltas.

So revision deltas become:

* sentence added/removed/changed,
* actor/action/object tuple deltas,
* claim delta,
* attribution delta,
* anchor delta,
* event candidate delta.

Then, secondarily:

* timeline-visible delta,
* graph materiality score,
* reviewer packet.

That makes revision a projection over **state differencing**, not a half-timeline subsystem.

In your context, **AAO = Actor–Action–Object**.

It’s just a compact way of representing the core structure of a sentence or clause as a **triple**:

```text
Actor → Action → Object
```

---

# What each part means

### 1. Actor

Who is doing the thing

* subject of the sentence (or inferred subject)

Examples:

* “**John** ate the apple” → Actor = John
* “**He** ran” → Actor = He (resolved to prior entity)

---

### 2. Action

What is happening

* the predicate / verb (often normalised)

Examples:

* “ate”, “ran”, “signed”, “was arrested”

---

### 3. Object

What the action is applied to (if any)

* direct object, indirect object, or target

Examples:

* “apple” in “ate the apple”
* “contract” in “signed the contract”
* may be empty:

  * “He ran” → no object

---

```text
→ Action: acquire
→ Object: startup
```

# 1) Deterministic extraction rules (dependency-based)

Assume a standard dependency parse (spaCy / Universal Dependencies style).

## Core idea

* **Event = verb head**
* **Everything else = attached via dependency edges**

---

## A. Find event heads (verbs)

Select tokens where:

* `pos_ == VERB` OR `pos_ == AUX` (filtered)
* OR `dep_ == ROOT` and behaves like predicate

```python
def is_event_head(tok):
    return (
        tok.pos_ in {"VERB", "AUX"}
        and tok.dep_ not in {"aux", "cop"}  # exclude helpers
    )
```

---

## B. Extract Actor (subject)

Priority:

* `nsubj`
* `nsubjpass`
* fallback: inherit from previous coordinated clause

```python
def get_actor(verb):
    for child in verb.children:
        if child.dep_ in {"nsubj", "nsubjpass"}:
            return resolve_coref(child)
    return inherit_actor(verb)
```

---

## C. Extract Object

Look for:

* `dobj` (direct object)
* `pobj` (object of preposition)
* `attr` (copular complement)

```python
def get_object(verb):
    for child in verb.children:
        if child.dep_ in {"dobj", "attr"}:
            return child
    return None
```

---

## D. Extract modifiers (THIS is your key upgrade)

### 1. Temporal modifiers

Dependencies:

* `prep` (prepositions like “after”, “before”)
* `advmod` (then, later)
* `npadvmod` (temporal noun phrases)

```python
TEMPORAL_PREPS = {"after", "before", "during", "since", "until"}

def get_temporal_mods(verb):
    mods = []
    for child in verb.children:
        if child.dep_ == "prep" and child.text.lower() in TEMPORAL_PREPS:
            pobj = next((c for c in child.children if c.dep_ == "pobj"), None)
            if pobj:
                mods.append({
                    "type": child.text.lower(),   # after/before
                    "reference": pobj.text
                })
    return mods
```

---

### 2. Adverbial sequence markers

```python
SEQUENCE_WORDS = {"then", "next", "later", "afterwards"}

def get_sequence_mods(verb):
    return [
        {"type": "sequence", "value": child.text}
        for child in verb.children
        if child.dep_ == "advmod" and child.text.lower() in SEQUENCE_WORDS
    ]
```

---

### 3. Coordination (splitting events)

This is critical for your “woke up, brushed teeth, went to work”.

Dependency:

* `conj`

```python
def split_conj_chain(verb):
    events = [verb]
    events.extend([c for c in verb.children if c.dep_ == "conj"])
    return events
```

# 3) Full extraction flow (drop-in pseudocode)

```python
def extract_events(doc):
    events = []

    for tok in doc:
        if not is_event_head(tok):
            continue

        # split coordination
        verbs = split_conj_chain(tok)

        for i, verb in enumerate(verbs):
            actor = get_actor(verb)
            obj = get_object(verb)

            temporal = get_temporal_mods(verb)
            sequence = get_sequence_mods(verb)

            events.append({
                "actor": actor,
                "action": verb.lemma_,
                "object": obj.text if obj else None,
                "temporal_relations": temporal,
                "sequence_markers": sequence,
                "clause_index": i,
                "source_span": verb.sent.text,
            })

    return events
```
